What is Illusion?

Illusion is an opinionated, mobile-first, progressive enhanced, accessible, SCSS base kit to kickstart websites with.

Illusion is built to be used on top of Modernizr and Normalize. When included, Illusion does nothing. Enable features by setting SCSS variables.

Features

  • Grid mixins
  • 8 point grid
  • Accessible forms
  • Useful functions
  • Useful mixins

Why use Illusion?

Write consistent and clean code in less time. Based on an 8 point grid for harmonized design

Getting started

Bower install, or NPM, or whatever

Include Modernizr and add at least JS - No JS detection and flexbox detection if you're gonna use flexbox

Include Normalize and include it before anything else

Define your variables before loading illusion

Add your own mixins after illusion

Grid mixins

One of the most important aspects of a website is the grid is based on. It can make a page feel balanced and in harmony when used right, but when used wrong a page can look very untrustworthy.

Illusion uses a grid that:

  1. Is based on an 8 point grid
  2. Has responsive gutters ranging from 16px to 40px
  3. Is using calc() and pixels so vertical gutters follow horizontal ones
  4. Comes with an optional fallback for the body element to have a set-width

Grid mixins:

Container

Use for outer containers only. Add clearfix when used without flexbox and removes clearfix when used as flexbox.

Arguments
Name Type Default Description
flexbox string false Can be anything but false to work. For example both @include container(flex); and @include container(true); will enable flexbox
Dependency

For flexbox to work properly it uses Modernizr .flexbox.flexwrap

SCSS
            .foo {
              @include container();
            }
            .bar {
              @include container(flex);
            }
CSS
            .foo {
                margin-left: auto;
                margin-right: auto;
                max-width: 1280px;
                padding-left: 16px;
                padding-right: 16px;
            }

            .foo:before,
            .foo:after {
                display: table;
                content: " ";
            }

            .foo:after {
                clear: both;
            }

            .flexbox.flexwrap .bar {
                display: flex;
                flex-wrap: wrap;
                // And more...
            }

The 8 point grid

By placing everything on an 8 point grid you ensure that your design looks grisp on all screens. Even screens with a 1.5 dpi.

Another great advantage is that your whole design looks very balanced.

Exceptions coul be fonts and their line-heights. For an optimal result you should try to maintain the line-height on the 8pt grid as much as possible. Whenever that's impossible you can always fall back to 4 pixel instead of. You can have a line-height of 20px for example.

How to maintain your 8pt grid

For the grid mixins everything is in the 8pt grid.

Paragraphs, headings and other styled block elements there's a standard 24px bottom with an exception for the headings. They have 16px to form groups with their content below.

For all other elements we have two more options to use

  1. Use our spacing mixin to follow the responsive gutter.
  2. If you don't want to follow the responsive gutter you can use our spacing function.

Spacing mixin

The mixin follows the responsive breakpoints so you don't have to worry about it yourself. You can also stop the 'growing' at any breakpoint (or custom width), it will keep the spacing from that breakpoint. It also accepts 0.5, 1 and 2 as an argument for the amount of spacing.

Spacing function

Accepts a value of 0.5, 1 or 2. If used in @include breakpoint it uses that current gutter. Also possible to define which 'size' it should use.
